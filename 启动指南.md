# AgentHub 项目启动指南

## 环境要求

| 工具      | 版本      | 检查命令               |
| ------- | ------- | ------------------ |
| Node.js | >= 18   | `node -v`          |
| pnpm    | 最新      | `pnpm -v`          |
| Python  | >= 3.10 | `python --version` |

后端依赖：`fastapi uvicorn httpx pydantic python-dotenv cryptography qrcode pillow resend`
测试依赖：`pytest`

```bash
pip install -r requirements.txt
pip install pytest   # 仅开发期
```

## 1. 配置 .env（首次启动必做）

```bash
cd /d F:\A-项目\13-中台网站
cp .env.example .env
```

编辑 `.env`，**至少修改 2 项**：

```bash
# 必填：用 Python 生成 32 字节随机盐
EMAIL_HASH_SALT=replace-with-output-of-secrets.token-hex-32

# 强烈推荐 dev 模式（链接打到 FastAPI 日志，不真发邮件）
DEV_EMAIL_MODE=1
```

生成 `EMAIL_HASH_SALT`：

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

> **生产环境**（Netlify + 真实邮件）：`DEV_EMAIL_MODE=0`，并填齐：
> - `RESEND_API_KEY`（https://resend.com/api-keys）
> - `RESEND_FROM`（Resend 后台已验证的发件域名，例如 `AgentHub <noreply@yourdomain.com>`）
> - `APP_PUBLIC_BASE`（Netlify 公网 URL，用于邮件中拼接登录链接）

## 2. 初始化数据库（首次启动必做）

```bash
python scripts/init_credit_db.py
```

预期：

```
[ok] schema ensured at F:\A-项目\13-中台网站\data\accounts.db
```

幂等地创建 4 张表（保留已有 `accounts` 表）：

| 表 | 作用 |
|---|---|
| `users` | 用户身份（email_hash + masked email） |
| `sessions` | 登录态（HttpOnly cookie，30 天滚动） |
| `credit_accounts` | 积分余额（缓存层） |
| `credit_ledger` | 流水（事件溯源，追加不删） |
| `email_tokens` | 一次性登录 token（15 分钟过期） |

已有 `accounts` 表（平台账号绑定）**不会**被影响。

## 4. 访问地址

| 服务     | 地址                                    |
| ------ | ------------------------------------- |
| 前端     | <http://localhost:3000>               |
| 后端 API | <http://127.0.0.1:8000>                |
| API 文档 | <http://127.0.0.1:8000/docs>           |

API 路由：

| 路由 | 方法 | 用途 |
|---|---|---|
| `/api/auth/send-link` | POST | 发送登录邮件（body: `{email}`） |
| `/api/auth/verify-token` | POST | 验证 token 自动登录（body: `{token}`） |
| `/api/auth/logout` | POST | 退出登录 |
| `/api/auth/me` | GET | 当前用户 + 余额 |
| `/api/credit/balance` | GET | 余额查询 |
| `/api/credit/ledger` | GET | 流水查询 |

浏览器端登录入口：`/auth/verify?token=xxx` —— 邮件链接打开此页自动登录后跳回首页。

## 5. 浏览器验证（首次启动必做）

打开 <http://localhost:3000>：

1. **未登录状态**：顶部右上角有 `登录` 按钮，**左下角**侧边栏底部也有 `登录 / 注册` 入口
2. 点击任意登录入口 → 弹窗输入邮箱 → 点 `发送登录链接`
3. **dev 模式**：链接在 FastAPI 终端日志里（搜 `[DEV-EMAIL]`），形如 `http://localhost:3000/auth/verify?token=xxx`
4. 复制链接到浏览器 → 打开后自动验证 → 跳回首页
5. **登录成功后**：
   - 顶部右上角：`🪙 100 积分`（注册赠送）
   - 左下角用户卡片：邮箱脱敏（如 `a**e@example.com`）+ 余额
6. 点击左下角用户卡片 → 弹出 `退出登录` 菜单
7. **刷新页面**：登录态保持（cookie 30 天有效）
8. 重新登录：余额**不再增加**（同一邮箱不会重复赠送）

## 7. 故障排查

| 现象 | 解决 |
|---|---|
| 启动报 `EMAIL_HASH_SALT env var is required` | `.env` 没设该项或用了占位符。`python -c "import secrets; print(secrets.token_hex(32))"` 生成 |
| 启动报 `no such table: users` | 没跑 `python scripts/init_credit_db.py` |
| 顶部"登录"按钮看不到 | 浏览器 DevTools 看 Console；正常情况下应可见 |
| 邮件收不到（dev 模式） | 检查 FastAPI 终端 `[DEV-EMAIL]` 日志 |
| 邮件收不到（生产） | 检查 RESEND_API_KEY / RESEND_FROM / 域名验证 |
| 邮件链接打开后报"链接无效或已过期" | token 15 分钟过期；可能是用过的（一次性）。重新发 |
| 端口 3000/8000 被占 | 拆分启动方式，单独指定 `--port` |
| 登录后余额不显示 | 检查 `lib/fastapi-base.ts` 的 `proxyToFastapi` cookie 透传 |

## 8. 部署到生产（Netlify + Resend）

1. Netlify 后台设置环境变量（**不要带 `NEXT_PUBLIC_` 前缀的服务端变量**）：
   - `DEEPSEEK_API_KEY`、`ARK_*`
   - `EMAIL_HASH_SALT`、`RESEND_API_KEY`、`RESEND_FROM`、`APP_PUBLIC_BASE=https://your-domain.com`、`DEV_EMAIL_MODE=0`
2. FastAPI 部署到独立容器（Render / Railway / Fly.io），把 `NEXT_PUBLIC_FASTAPI_URL` 设为 FastAPI 公网地址
3. 部署后跑 `python scripts/init_credit_db.py` 创建生产库表
4. 部署后跑 `python scripts/reconcile.py` 验证余额无错位
5. `scripts/reconcile.py` 接 cron，每天跑一次对账
