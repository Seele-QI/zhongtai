# AgentHub 项目启动指南

## 环境要求

| 工具      | 版本      | 检查命令               |
| ------- | ------- | ------------------ |
| Node.js | >= 18   | `node -v`          |
| pnpm    | 最新      | `pnpm -v`          |
| Python  | >= 3.10 | `python --version` |
| pip 包   | 见下      | `pip list`         |

后端新增依赖：`aliyun-python-sdk-core`、`aliyun-python-sdk-dysmsapi`、`pytest`（仅测试）

```bash
pip install -r requirements.txt
pip install pytest   # 仅开发期跑测试需要
```

## 1. 配置 .env（首次启动必做）

```bash
cd /d F:\A-项目\13-中台网站
cp .env.example .env
```

编辑 `.env`，**至少修改两项**：

```bash
# 必填：用 Python 生成 32 字节随机盐
PHONE_HASH_SALT=replace-with-output-of-secrets.token-hex-32

# 强烈推荐开启 dev 模式（不真发短信，验证码打到 FastAPI 日志）
DEV_SMS_MODE=1
```

生成 PHONE_HASH_SALT 的命令：

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

> **生产环境**（Netlify + 真实短信）：`DEV_SMS_MODE=0`，并填齐以下 4 项：
> - `ALIYUN_SMS_ACCESS_KEY_ID`
> - `ALIYUN_SMS_ACCESS_KEY_SECRET`
> - `ALIYUN_SMS_SIGN_NAME`（阿里云控制台已审核的短信签名）
> - `ALIYUN_SMS_TEMPLATE_CODE`（短信模板编码）

如果想修改 Next 端口或 FastAPI 端口，新增：

```bash
# 改 Next 端口（默认 3000）
# pnpm dev:web -- --port 3100
# 改 FastAPI 端口（默认 8000）
# python -m uvicorn main:app --host 127.0.0.1 --port 8001
# Next → FastAPI 反代地址（默认 http://127.0.0.1:8000）
# NEXT_PUBLIC_FASTAPI_URL=http://127.0.0.1:8001
```

## 2. 初始化数据库（首次启动必做）

```bash
python scripts/init_credit_db.py
```

预期输出：

```
[ok] schema ensured at F:\A-项目\13-中台网站\data\accounts.db
```

这条命令会幂等地创建 5 张积分系统表：

| 表 | 作用 |
|---|---|
| `users` | 用户身份（phone_hash + masked phone） |
| `sessions` | 登录态（HttpOnly cookie，30 天滚动） |
| `credit_accounts` | 积分余额（缓存层） |
| `credit_ledger` | 流水（事件溯源，追加不删） |
| `sms_codes` | 短信验证码（5 分钟过期，5 次锁定） |

已存在的 `accounts` 表（平台账号绑定）**不会**被影响。

## 5. 浏览器验证（首次启动必做）

打开 <http://localhost:3000>：

1. **未登录状态**：顶部右上角有 `登录` 按钮，**左下角**侧边栏底部也有 `登录 / 注册` 入口
2. 点击任意登录入口 → 弹窗输入 11 位手机号 → 点 `获取验证码`（dev 模式下不发短信，验证码在 FastAPI 终端日志里）
3. 复制 6 位验证码到弹窗 → 点 `登录`
4. **登录成功后**：
   - 顶部右上角：`🪙 100 积分`（注册赠送）
   - 左下角用户卡片：头像（手机号末 2 位）+ 脱敏手机号 + 余额
5. 点击左下角用户卡片 → 弹出 `退出登录` 菜单
6. **刷新页面**：登录态保持（cookie 30 天有效）
7. 重新登录：余额**不再增加**（同一手机号不会重复赠送）

dev 模式下验证码的查找位置：

```bash
# 在 FastAPI 终端日志里搜
WARNING:sms:[DEV-SMS] phone=13800001234 code=123456
```

## 6. 常用命令

```bash
# 开发
pnpm dev          # 仅前端（端口 3000）
pnpm dev:web      # 同 dev
pnpm dev:api      # 仅后端（端口 8000）
pnpm dev:all      # 前端 + 后端（3000 + 8000）

# 数据库
python scripts/init_credit_db.py   # 初始化积分系统 5 张表（幂等）
python scripts/reconcile.py        # 对账：检查 balance == sum(ledger.delta)

# 测试
python -m pytest tests/ -v         # 17 个测试（auth 9 + credit 8）

# 构建 / 检查
pnpm build
pnpm lint
```

## 7. 故障排查

| 现象 | 解决 |
|---|---|
| 启动报 `PHONE_HASH_SALT env var is required` | `.env` 没设该项，或用了占位符。运行 `python -c "import secrets; print(secrets.token_hex(32))"` 替换 |
| 启动报 `operational error: no such table: users` | 没跑 `python scripts/init_credit_db.py` |
| 顶部"登录"按钮看不到 | 浏览器 DevTools 看 Console 是否有报错；正常情况下应可见 |
| 验证码收不到（dev 模式） | 检查 FastAPI 终端的 WARNING 日志 |
| 验证码收不到（生产模式） | 检查阿里云 SMS 4 项 env 是否齐全、模板是否审核通过、签名是否一致 |
| 端口 3000/8000 被占 | 用 3.2 节拆分启动方式，单独指定 `--port`；或 `netstat -ano \| findstr :3000` 找进程结束 |
| 登录后余额不显示 | `/api/credit/balance` 返回 401 是正常的 cookie 没透传——检查 `lib/fastapi-base.ts` 的 `proxyToFastapi` 是否带 cookie |

## 8. 部署到生产（Netlify）

当前项目已配置 `netlify.toml` + `pnpm-workspace.yaml`，部署流程：

1. 在 Netlify 后台设置环境变量（**不要带 `NEXT_PUBLIC_` 前缀的服务端变量**）：
   - `DEEPSEEK_API_KEY`（文案/对话）
   - `ARK_*`（方舟豆包）
   - `PHONE_HASH_SALT`、`ALIYUN_SMS_*`、`DEV_SMS_MODE=0`
2. FastAPI 部署到独立容器（Render / Railway / Fly.io），把 `NEXT_PUBLIC_FASTAPI_URL` 设为 FastAPI 公网地址
3. 部署后跑 `python scripts/init_credit_db.py` 创建表（生产库）
4. 部署后跑 `python scripts/reconcile.py` 验证余额无错位
5. 接入 `scripts/reconcile.py` 到 cron，每天跑一次做对账
