# 中台网站项目审计、一键分发核验与团队智能体统一设计

## 背景

本次工作覆盖三类目标：

1. Review 整体项目，梳理框架，并检查 DeepSeek 开发项目中的安全与可用性风险。
2. 检查并修复“一键分发”功能，确保在本地联调环境下可正常运行。
3. 更新“智能体团队”板块，将团队智能体统一为“人物 + 方法论/职业标签”风格，并接入所有团队聊天入口的智能体选择。

本次采用方案 B：统一智能体注册表 + 定点修复一键分发 + 安全审计与自主验证。

## 当前现状

### 架构现状

- 主应用为 Next.js 16 App Router，业务主控在 `app/page.tsx`。
- AI 对话主网关为 `app/api/ai/chat-stream/route.ts`，文本与多模态请求会转发至 DeepSeek 或火山方舟。
- 视频生成与一键分发依赖可选 FastAPI 服务 `main.py`。
- 团队智能体数据当前分散在 `components/dashboard-view.tsx` 与 `components/agent-center.tsx` 两处，存在重复定义与内容漂移风险。
- 团队聊天 `components/chat-workspace.tsx` 目前只支持“打开指定团队智能体”，未内建团队智能体切换器。
- 文案聊天 `components/copywriting-chat-workspace.tsx` 已具备智能体选择器，但数据源是单独的文案智能体列表。

### 已识别风险

- 一键分发前端硬编码 `http://127.0.0.1:8000`，跨环境不可用。
- 分享令牌仅保存在 FastAPI 进程内存中，服务重启后链接失效。
- FastAPI CORS 为全开放。
- Next 构建启用了 `ignoreBuildErrors: true`，会掩盖 TypeScript 构建错误。
- DeepSeek / 方舟接口依赖环境变量，需核查缺省分支、异常透传与边界处理是否合理。

## 本次范围

### 包含

- 输出项目结构与风险审计结论。
- 修复并验证一键分发核心链路。
- 抽离统一的团队智能体注册表。
- 让工作台首页、智能体中心、团队聊天选择器共用同一份团队智能体数据。
- 将团队智能体改为“人物名 + 角色 + 标签”的展示形式。
- 对改动后的关键链路做实际启动与功能验证。

### 不包含

- 全量重构文案智能体体系。
- 对外部平台抖音创作者中心进行自动化登录或真实表单注入。
- 引入完整权限系统、数据库迁移或线上部署方案重构。

## 设计目标

### 目标一：统一团队智能体数据源

新增一个共享的团队智能体注册表模块，保存：

- 唯一标识
- 展示名
- 人物化角色标题
- 描述
- 标签
- 头像
- 在线状态
- 聊天欢迎语
- 快捷提问

该注册表作为单一数据源，供以下位置消费：

- `DashboardView` 的“智能体团队”
- `AgentCenter` 的“我的智能体团队”
- `ChatWorkspace` 的团队智能体切换器
- 可能的全局搜索或其它团队入口

### 目标二：修复一键分发链路

一键分发功能修复以“本地联调可真实走通”为准，包含：

- 前端不再硬编码 FastAPI 地址，而是统一读取可配置的服务端基地址。
- 生成分享链接、生成二维码、点击分享链接跳转这三段链路可以在本地验证。
- 对分享失败、后端缺失、令牌失效等情况给出可读错误信息。

### 目标三：完成项目 review 与 DeepSeek 风险检查

在代码层输出结构化结论，按严重度识别：

- 会导致不可用的问题
- 安全面风险
- 部署/维护性风险
- 已修复与未修复项

## 方案设计

### 1. 团队智能体注册表

新增一个共享数据模块，例如 `lib/team-agents.ts`，定义统一类型：

- `id`
- `name`
- `role`
- `description`
- `avatar`
- `tags`
- `status`
- `themeColor`
- `quickPrompts`

人物内容统一改造成“现实人物 / 思维模型 / 专业身份”的风格，例如：

- 查理·芒格 - 多元思维模型，投资分析师
- 彼得·德鲁克 - 管理创新，组织顾问
- 纳瓦尔 - 杠杆思维，个人商业顾问

实现原则：

- 首页与中心页只负责展示，不再各自内嵌数据。
- 团队聊天选择器从注册表读取所有团队智能体。
- 打开聊天时透传角色、头像、主题色与快捷提问。

### 2. 团队聊天选择器

扩展 `ChatWorkspace`：

- 增加可选的 `allAgents` 与 `onAgentSwitch` 属性。
- 当传入团队智能体列表时，在顶部展示切换入口，交互风格向现有 `CopywritingChatWorkspace` 靠齐。
- 切换智能体时，按智能体名隔离本地存储 key，避免串会话。

`app/page.tsx` 负责把统一的团队智能体列表传给 `ChatWorkspace`，并在切换时同步更新当前激活智能体元数据。

### 3. 一键分发修复

新增一个统一的后端基地址读取逻辑，例如复用环境变量工具或新增前端常量模块：

- 本地默认指向 `http://127.0.0.1:8000`
- 支持通过环境变量覆盖

前端 `ShareDistribute` 的两类请求统一走该地址：

- `/api/agent/chat`
- `/api/share/generate`

同时补充：

- 请求超时或后端不可达时的错误提示
- JSON 解析失败时的回退提示
- 分享成功后二维码与复制链接的手动验证

### 4. 项目审计产物

项目审计结果不单独做成新页面，而是在最终交付说明中输出：

- 架构结论
- 风险清单
- 已修复项
- 残余风险

若实现中发现适合沉淀为仓库文件，可补充一个 Markdown 报告。

## 验证设计

### 静态验证

- 对改动文件执行类型/诊断检查。
- 确认没有引入明显的 JSX/TS 语法错误。

### 运行验证

启动：

- `pnpm dev:all`

验证项：

1. 工作台首页“智能体团队”卡片均已替换为新人物化智能体。
2. 智能体中心列表与首页一致。
3. 从首页或中心页打开团队聊天后，可以在聊天顶部切换到其它团队智能体。
4. 切换后头像、角色、快捷提问与会话隔离行为正确。
5. 一键分发可完成：
   - 选择视频
   - AI 自动填写或手动填写
   - 生成分享链接
   - 生成二维码
   - 打开分享链接并跳转
6. DeepSeek 聊天主链路未被此次修改破坏。

### 结论表达

最终结果按三类给出：

- 已验证通过
- 已修复但受第三方平台限制
- 未纳入本次修复范围

## 风险与边界

- 分享令牌仍为内存存储，本地演示可用，但进程重启后会失效；这是已知边界。
- 如果抖音创作者中心禁止第三方页面注入本地存储数据，则“一键带参跳转”只能保证跳转，不保证目标站读取参数。
- 如果本地未配置有效 DeepSeek Key，聊天链路无法完成真实验证，但前端错误提示应保持可用。

## 实施顺序

1. 抽离统一团队智能体注册表。
2. 接入首页与智能体中心。
3. 为团队聊天接入智能体选择器。
4. 修复一键分发地址与异常处理。
5. 审计 DeepSeek 与项目风险并同步修复可修项。
6. 启动项目，执行完整验证并整理结论。

## 完成定义

满足以下条件才视为完成：

- 三项任务均有代码或审计结果交付。
- 团队智能体人物化改造已统一落地。
- 一键分发在本地联调环境下可以成功生成分享链接和二维码并完成跳转。
- 关键功能已自主验证，最终说明中明确记录验证结果与残余风险。
